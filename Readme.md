# Плагин: ModGuard Bypass 🛡️

**ModGuard Bypass** — это интеллектуальный и гибкий плагин для панели BlockMine, который позволяет вашим ботам беспрепятственно обходить проверки модов при подключении к серверам с Forge или Fabric модификациями.

Благодаря встроенной технологии **«Зеркального обхода» (Mirror Bypass)**, плагин самостоятельно перехватывает запросы серверов, анализирует списки требуемых модов и отправляет точно сбалансированный ответ, избавляя вас от необходимости прописывать сотни модов вручную.

---

## Ключевые возможности

*   **⚡ Умный авторежим (`auto`)**: Бот автоматически анализирует поток сетевых пакетов при входе. Если сервер запрашивает Forge-проверку — включается обход Forge. Если Fabric — включается обход Fabric. Лишние настройки при этом скрываются из веб-интерфейса панели.
*   **⚙️ Ручной режим (`custom`)**: Позволяет вручную зафиксировать лоадер и прописать кастомную строку бренда клиента для обхода нестандартных защит.
*   **📑 Полное логирование модов**: Если в настройках включен параметр *«Подробное логирование модов»*, бот выведет в вашу веб-консоль аккуратную таблицу со списком всех модов и их версий, которые установлены на сервере.
*   **🔄 Поддержка всех версий протокола**:
    *   *Legacy (например, 1.12.2)* — эмулирует весь цикл рукопожатия `FML|HS` (ServerHello, ClientHello, ModList, HandshakeAck).
    *   *Modern (1.13 - 1.21+)* — обходит запросы Forge/Fabric через логин-плагины (`login_plugin_request`).

---

## Доступные настройки в панели управления

*   **Режим конфигурации (`actionsPreset`)**:
    *   `Автоматически`: Плагин сам определяет лоадер сервера и настраивает бренд. Все сопутствующие ручные настройки скрываются.
    *   `Вручную`: Открывает ручные настройки бренда и лоадера.
*   **Мод-лоадер сервера (`enableModLoader`)**: Выбор между `Forge` и `Fabric` (виден только в ручном режиме).
*   **Кастомный бренд клиента (`enableCustomBrand`)**: Позволяет отправить серверу любой бренд (например, `forge`, `fabric`, `fml`, `vanilla`) (виден только в ручном режиме).
*   **Подробное логирование модов (`verboseLogging`)**: Включает/выключает детальный вывод информации о модах сервера в консоль.

## Структура файлов плагина
Плагин спроектирован по модульному принципу, что гарантирует его стабильность и читаемость кода:

```
/plugins/mod-guard-bypass
├── lib/
│   ├── ModHandshakeHandler.js   # Модуль сетевого обхода
│   └── ModPacketParser.js       # Декодер сетевых пакетов списка модов
├── index.js                     # Точка входа в плагин BlockMine
├── package.json                 # Настройки и манифест плагина
└── README.md                    # Эта документация
